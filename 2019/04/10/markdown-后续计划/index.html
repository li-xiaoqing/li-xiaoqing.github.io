markdown-后续计划

发表于 2019-04-10 | 分类于 技术 | |

总体目标

markdown编辑器能够高效稳定的支撑内部业务方使用。同时打造出一款比市场上功能齐全且方便使用的编辑器，做好技术优化重构，完成开源并推广。

目标分解

修复问题&补充功能，优化体验

  • 首先对我们自己的编辑器做功能的查漏补缺，比如我们编辑器现在缺少流程图渲染的支持、缺少键盘快捷键输入、无输入区域和渲染区域的高度同步、无阅读模式和历史记录前进后退等功能；
  • 其次需要再次体验其它编辑器，并做好横向对比，分析优劣，对比功能多样性，性能高低，是否容易扩展等；
  • 最后就是需要在使用的过程中，不断的思考交互和体验，比如目前还没有快捷输入hover提示、弹窗关闭逻辑不太优雅等。

开源

为项目定一个名字perfect-markdown, 通过github开源，如果事件允许，可能会同时开源一款更强大的桌面版的软件。

完成2.0版本重构

从技术栈、可扩展性、插件化和配置化四方面思考2.0版本。

推广

是一个长期需要考虑的事情，首先是做大做强，然后是github推广，相关站点seo等等

本文结束感谢您的阅读
